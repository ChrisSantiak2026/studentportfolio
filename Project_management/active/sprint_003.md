# Sprint 003- Styling & Responsiveness

## Goal

 Apply CSS rules to transform the raw text into a professional, readable, and mobile-friendly design.

## Scope (In/Out)

IN:
- Typography scaling, monochrome color palette (with one accent color), whitespace enforcement, and a mobile media query.

OUT: 
- CSS animations 
- complex grid systems
- hover effects (beyond basic link underlines).

## Tasks

- Import a clean sans-serif font (e.g., Inter) from Google Fonts.

- Set global background and text colors (white background, dark grey text).

- Apply max-width: 65ch to paragraphs to enforce readable line lengths.

- Add margin-bottom: 80px to the main semantic sections to create visual breathing room.

- Write a @media (max-width: 768px) query to ensure the layout stacks neatly on smaller screens.

## Files to touch

- style.css

## Acceptance criteria

- The site possesses a clear visual hierarchy, is easy to read on a desktop monitor, and adapts flawlessly to a mobile viewport.

## Verification checklist

-[ ] A maximum of two font families are used.
-[ ] Only one accent color is present (used exclusively for links).
- [ ] There is a distinct, empty visual gap separating the Hero, Projects, and Footer sections.
- [ ] Shrinking the browser window to mobile width introduces zero horizontal scrolling.

## Completion evidence

- A fully populated style.css file linked to your HTML that completely transforms the browser view into a structured, well-spaced, and cleanly styled digital document.
