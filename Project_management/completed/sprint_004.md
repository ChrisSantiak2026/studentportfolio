# Sprint 004-  Deployment & Final QA 
## Goal

- Push the local codebase to a public GitHub repository and configure GitHub Pages for live hosting.

## Scope (In/Out)

IN:
- Git commit 
- remote repository creation 
- code push 
- GitHub Pages activation
- live environment testing.

OUT: 
- Custom domain name configuration, 
- DNS record management

## Tasks

- Stage and commit all final changes using git add and git commit.

- Create a new public repository on GitHub named [username].github.io.

- Link the local repository to the remote URL and run git push -u origin main.

- Navigate to the repository settings on GitHub and enable Pages from the main branch.

- Perform a final QA check on the live URL.

## Files to touch

- Local Terminal

- GitHub Web Interface

- Acceptance criteria: The portfolio is live, publicly accessible on the internet, and functions exactly as it did in the local environment.

## Verification checklist

- [ ] The website successfully loads at https://[username].github.io.

- [ ] All project links and contact links route correctly in the live environment.

- [ ] The live site passes the mobile responsive check on an actual smartphone.

- Completion evidence: A live, shareable URL that hosts the finished portfolio website, ready to be sent directly to recruiters.
